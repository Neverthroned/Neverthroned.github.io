Invection 5: Eye VFX

By using concepts in this video I made VFX for a boss enemy

The video above by A Bit of Game Dev demonstrates the implementation of a shader for an explosion effect, which I appropriated in order to create a boss enemy themed around an Eye.

Image 1: This is the first part of the effect, largely unmodified compared to the video tutorial linked above. By taking a noise texture and applying a fresnel effect to it you can create interesting VFX when applying the shader to a mesh. This section of the graph implements the 'dark color' of the effect, and in this case the Eye's outline. This section also is connected to an add node to be combined with the light color and the alpha of the graph which allows for transparency.

Image 2: This section of the graph is largely similar to the first section, with the only differences being the power of the fresnel effect and the saturate node. This creates the light color, or the pupil of the eye, and is linked to the add node from before and the emission of the graph, allowing for a more pronounced light effect.

Image 3: The shader graph makes heavy usage of noise in order to achieve the desired effect. The inner portion of the eye uses a standard noise texture plugged into both of the texture samples from before, however the outer eye is a bit more complicated. For that I used a video file created from TouchDesigner which required creating a video player object within the Unity Editor. This video player would cast to a render texture to be applied to an identical shader graph as before within those texture samples. This would create the animated effect I was looking for. Within this image, I'm creating the first noise insert in TouchDesigner from scratch. That is to say this is where I divert from the tutorial video's instructions.

Image 4: The previous noise insert is fed into this graph to to create several glitch effects, though the most notable is what many would refer to as circuit bending. After all of this, its fed into a movie file out to be used in Unity, more specifically to be implemented into the video player object from before.

Final Render: The final effect renders a glitchy looking Eye, with an inner and outer shell


Explanation: While the video detailed using this practice in the form of an explosion that also makes usage of the Unity particle system I opted to instead use the concept of the fresnel effect in order to achieve a desirable VFX for a boss character. This required heavy improvisation on my part in order to get this to properly work. Completely contrary to the video, I instead wanted to create a standalone effect that would render an intimidating and unique boss, using outside software such as TouchDesigner to get the noise effects I wanted. These noise effects would then be integrated into the video player system to then be cast into a texture usable by the shader graph. Ultimately, a lot of this is mainly about aesthetic choice, and manipulating what kind of surface the different portions of the Eye should render as. The inner portion acting as a solid, opaque sphere with slight alpha clipping, and the outer as a transparent object with largely opposing settings for dissolve and smoothstep. The next steps for me would to integrate visual cues for when the eye plans to attack by animating the integral settings.
